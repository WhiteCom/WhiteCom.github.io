## <span class="text-orange">C++ Undefined Behavior(UB)</span>

[ Undefined Behavior 정의 ]

"이 코드는 결과가 어떻게 나와도 컴퓨터언어 표준이 책임지지 않는다."

즉 해당 동작이 정의되어있지 않은 상태를 Undefined Behavior (UB) 라고 합니다. 

UB 상태가 되면, 아래와 같은 결과를 산출한다.

1. 컴파일, 실행까진 되나 이상한 결과가 나옴

2. 크래시가 발생함

<span class="text-orange">3. 컴파일러마다 다른 결과가 출력됨.</span>

4. 기타 등등...

아래는 대표적인 UB 예시를 나타낸 것입니다. 

---

### 1. 배열 범위 초과

```cpp
{
    int arr[3] = {1, 2, 3};
    arr[5] = 10; // UB
}
```

---

### 2. 초기화 안 된 변수

```cpp
{
    int x;
    std::cout << x << endl; // UB
}
```

---

### 3. 같은 변수 1개 라인에서 증감연산자 사용

```cpp
{
    int a = 5;
    std::cout << a++ + ++a; // UB
}
```

---

### 4. 잘못된 포인터 접근

```cpp
{
    int* p = nullptr;
    *p = 10; // UB
}
```

---

### 5. Strict Aliasing / Union 문제

```cpp
{
    float f = 1.0f;
    int* p = (int*)&f; // UB
}
```

---

### UB 가 무서운 이유

- 디버그 옵션 컴파일 + 실행 정상동작 존재
- 릴리즈 옵션 컴파일 + 실행의 경우 버그발생 및, 크래시 발생 존재
- 컴파일러 바꿀 경우 비정상동작 존재

ex)
```cpp
{
    int x = 0;
    
    if(x + 1 > x)
    {
        // 컴파일러가 항상 true 라고 처리한다고 가정
    }
}
```

따라서, 아래와 같은 경우 UB 는 문제가 되기에 반드시 피해야 함.

<span class="text-orange">1. 멀티스레딩</span>

<span class="text-orange">2. 메모리관리</span>

<span class="text-orange">3. 네트워크 I/O + 패킷 송수신</span>

<span class="text-orange">4. 실시간 렌더링</span>
